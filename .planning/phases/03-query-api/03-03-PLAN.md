```sql
weather_readings (time TIMESTAMPTZ, feature_id UUID, temperature FLOAT, dew_point FLOAT,
```

TimescaleDB last() aggregate (returns most recent non-NULL value):
```sql
SELECT last(aqi, time) AS current_aqi FROM air_quality_readings WHERE feature_id IN (...)
-- Always filter to 24-hour window to avoid scanning all chunks
```

CRITICAL pitfall: TimescaleDB time range queries MUST include both start and end bounds as Python datetime objects (not strings). asyncpg handles datetime → timestamptz correctly. Always cap query range at 90 days maximum.

Create backend/app/routers/timeseries.py implementing GET /api/timeseries/{domain}.

**Query parameters:**
- `domain` — path parameter
- `town` — query param (validated against current_town.id)
- `start` — ISO 8601 datetime string (parse with `datetime.fromisoformat` or use FastAPI's `datetime` param type)
- `end` — ISO 8601 datetime string
- Both are required. FastAPI can parse `datetime` query params directly with type annotation.

**Validation order:**
1. town != current_town.id → HTTPException(404)
2. domain not in VALID_DOMAINS → HTTPException(404)
3. end - start > 90 days → HTTPException(400, "Time range exceeds 90-day maximum")
4. start >= end → HTTPException(400, "start must be before end")

**Air quality query** (domain == "air_quality"):
```python
result = await db.execute(
    text("""
        SELECT
            r.time          AS time,
            r.feature_id::text AS feature_id,
            r.pm25, r.pm10, r.no2, r.o3, r.aqi
        FROM air_quality_readings r
        JOIN features f ON r.feature_id = f.id
        WHERE f.town_id = :town_id
          AND r.time >= :start
          AND r.time <  :end
        ORDER BY r.time ASC
        LIMIT 10000
    """),
    {"town_id": current_town.id, "start": start, "end": end},
)
rows = result.mappings().all()
```

Then map each row to `TimeseriesPoint(time=row["time"], feature_id=row["feature_id"], values={...})` where values contains the non-None numeric fields.

**Weather query** (domain == "weather"):
```python
result = await db.execute(
    text("""
        SELECT
            r.time, r.feature_id::text AS feature_id,
            r.temperature, r.wind_speed, r.condition, r.icon,
            r.observation_type
        FROM weather_readings r
        JOIN features f ON r.feature_id = f.id
        WHERE f.town_id = :town_id
          AND r.time >= :start
          AND r.time <  :end
        ORDER BY r.time ASC
        LIMIT 10000
    """),
    {"town_id": current_town.id, "start": start, "end": end},
)
```

**Other domains** (transit, water, energy — tables exist but may be empty in Phase 3): Return empty `TimeseriesResponse` with `count=0` and `points=[]`. Do not error on empty — these domains will be populated in later phases.

**Attribution**: Query sources table for `WHERE town_id = :town_id AND domain = :domain`. Map connector_class to CONNECTOR_ATTRIBUTION. Return as `list[dict[str, str]]` (not Attribution model — matches TimeseriesResponse.attribution type).

**last_updated**: MAX(r.time) from the result rows, or None if empty.

**Response assembly:**

**Part A — Create backend/app/routers/kpi.py:**

The KPI endpoint aggregates current values across domains into a single response.

**Air quality KPI** — use TimescaleDB `last()` aggregate over last 24 hours:
```python
aq_result = await db.execute(
    text("""
        SELECT
            last(aqi,  time) AS current_aqi,
            last(pm25, time) AS current_pm25,
            last(pm10, time) AS current_pm10,
            last(no2,  time) AS current_no2,
            last(o3,   time) AS current_o3,
            MAX(time)        AS last_updated
        FROM air_quality_readings r
        JOIN features f ON r.feature_id = f.id
        WHERE f.town_id = :town_id
          AND r.time > NOW() - INTERVAL '24 hours'
    """),
    {"town_id": current_town.id},
)
aq_row = aq_result.mappings().first()
```

Then call `tier, color = aqi_tier(aq_row["current_aqi"] if aq_row else None)`.

**Weather KPI** — most recent current observation (observation_type = 'current'):
```python
wx_result = await db.execute(
    text("""
        SELECT
            last(temperature, time) AS temperature,
            last(condition,   time) AS condition,
            last(wind_speed,  time) AS wind_speed,
            last(icon,        time) AS icon,
            MAX(time)               AS last_updated
        FROM weather_readings r
        JOIN features f ON r.feature_id = f.id
        WHERE f.town_id    = :town_id
          AND r.observation_type = 'current'
          AND r.time > NOW() - INTERVAL '48 hours'
    """),
    {"town_id": current_town.id},
)
wx_row = wx_result.mappings().first()
```

**Transit KPI** — count stops and route shapes from features table:
```python
tr_result = await db.execute(
    text("""
        SELECT
            COUNT(*) FILTER (WHERE source_id LIKE 'stop:%') AS stop_count,
            COUNT(*) FILTER (WHERE source_id LIKE 'shape:%') AS route_count,
            MAX(created_at) AS last_updated
        FROM features
        WHERE town_id = :town_id
          AND domain  = 'transit'
    """),
    {"town_id": current_town.id},
)
tr_row = tr_result.mappings().first()
```

NOTE: The FILTER pattern assumes GTFSConnector prefixes source_id with "stop:" and "shape:". Read backend/app/connectors/gtfs.py briefly to confirm the actual source_id prefix format used. Adjust the LIKE patterns to match the actual format. If uncertain, use a simpler approach: COUNT(*) with domain='transit' as stop_count and 0 as route_count.
