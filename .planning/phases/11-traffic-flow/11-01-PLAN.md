       - `lat`, `lon`: sample point coordinate on that road segment

       Sample points (approximate, along major Aalen arterials):
       B29 (east-west through Aalen): ~12 points from Essingen (lon 10.02) to Aalen-Ost (lon 10.15)

    2. TomTom Flow Segment Data API endpoint:
       `https://api.tomtom.com/traffic/services/4/flowSegmentData/absolute/10/json?point={lat},{lon}&key={api_key}&unit=KMPH`
       - style=absolute, zoom=10
       - Returns JSON with `flowSegmentData.currentSpeed`, `flowSegmentData.freeFlowSpeed`, `flowSegmentData.confidence`, `flowSegmentData.coordinates.coordinate` (list of {latitude, longitude} forming the segment LineString)

       - For each segment in AALEN_ROAD_SEGMENTS, make GET request to TomTom API
       - Use httpx.AsyncClient with timeout=30.0
       - Collect responses into a list of (segment_def, api_response_dict) tuples
       - Skip segments where the API returns an error (log warning, continue)
       - Return the list of tuples

    4. normalize() method:
       - For each (segment_def, response_data):
         - Extract `flowSegmentData.currentSpeed` and `flowSegmentData.freeFlowSpeed`
         - Compute `congestion_ratio = currentSpeed / freeFlowSpeed` (guard div-by-zero)
         - Map ratio to congestion_level: >= 0.75 -> "free", 0.50-0.75 -> "moderate", 0.25-0.50 -> "congested", < 0.25 -> "congested"
         - Create Observation with:
           - feature_id from self._feature_ids[segment_def["id"]]
           - domain="traffic"
           - values={"vehicle_count_total": None, "vehicle_count_hgv": None, "speed_avg_kmh": currentSpeed, "congestion_level": congestion_level}
           - NOTE: Map congestion_ratio into speed_avg_kmh and congestion_level to match existing traffic_readings schema. Store congestion_ratio and freeflow in feature properties instead.

    5. run() override (follow BastConnector pattern):
       - Call fetch() to get raw data
       - For each (segment_def, response_data): extract LineString coordinates from `flowSegmentData.coordinates.coordinate`, build WKT LINESTRING, call upsert_feature() with:
         - source_id=f"tomtom:{segment_def['id']}"
         - domain="traffic"
         - geometry_wkt=LINESTRING WKT
         - properties={"road_name": segment_def["name"], "segment_id": segment_def["id"], "data_source": "TomTom", "freeflow_kmh": freeFlowSpeed, "congestion_ratio": congestion_ratio, "confidence": confidence, "attribution": "TomTom Flow API"}
       - Cache feature IDs in self._feature_ids
       - Call normalize() then persist() then _update_staleness()

    6. Add helper methods:
       - `_is_rush_hour() -> bool`: Check current time in Europe/Berlin timezone. Rush = 06:00-08:59 or 16:00-18:59.
       - `_get_poll_interval() -> int`: Return 600 if rush, 1800 if off-peak.
