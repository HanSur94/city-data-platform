  <action>
    HeatDemandConnector (backend/app/connectors/heat_demand.py):
    - Inherit BaseConnector. Since actual KEA-BW GeoPackage download is not available, generate simulated building-level heat demand data.
    - fetch(): Generate ~50 simulated building points within Aalen bbox (9.97-10.22 lon, 48.76-48.90 lat) with random heat_demand_kwh_m2_y values between 30-350. Use deterministic seed based on town ID for reproducibility.
    - normalize(): For each building, compute heat_class based on kWh/m2/y thresholds from CONTEXT.md: blue <50, light_blue 50-100, green 100-150, yellow 150-200, orange 200-250, red >250. Domain = "energy" (reuse existing domain, feature_type="heat_demand" in properties to distinguish).
    - run(): upsert_feature for each building with POINT geometry, then persist observations, then _update_staleness.
    - Properties include: heat_demand_kwh_m2_y, heat_class, feature_type="heat_demand", source="KEA-BW Waermeatlas (simulated)".

    CyclingInfraConnector (backend/app/connectors/cycling.py):
    - Inherit BaseConnector. Follow OverpassCommunityConnector pattern.
    - fetch(): Query Overpass for ways with highway=primary|secondary|tertiary|cycleway plus cycling tags (cycleway, bicycle, cycleway:left, cycleway:right) within town bbox.
    - Overpass query: [out:json][timeout:60]; way["highway"~"primary|secondary|tertiary|residential|cycleway"]["bicycle"!~"no"]({bbox}); out geom;
    - normalize(): For each way, determine infra_type from OSM tags:
      - highway=cycleway -> "separated"
      - cycleway=lane OR cycleway:left=lane OR cycleway:right=lane -> "lane"
      - cycleway=advisory OR cycleway:left=advisory OR cycleway:right=advisory -> "advisory"
      - cycleway=shared_lane OR bicycle=designated -> "shared"
      - none of above on primary/secondary/tertiary -> "none"
      - skip residential roads with no cycling tags
    - Domain = "infrastructure", feature_type="cycling" in properties.
    - run(): upsert_feature for each way with LINESTRING geometry from Overpass geom, then persist, then _update_staleness.
  </action>
