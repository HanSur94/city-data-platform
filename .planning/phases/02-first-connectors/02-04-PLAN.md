  <behavior>
    GTFSConnector behavior:
    - fetch() downloads bwgesamt.zip from `self.config.config["gtfs_url"]` using httpx; returns raw bytes
    - fetch() uses `httpx.AsyncClient(timeout=120.0)` — GTFS zip is ~50MB
    - fetch() raises ValueError if response bytes are empty (len == 0)
    - normalize(raw: bytes) parses with `gtfs_kit.read_feed(io.BytesIO(raw), dist_units="km")`
    - normalize() IMMEDIATELY applies bbox filter after reading feed:
      ```python
      bbox = self.town.bbox
      stops = feed.stops[
          feed.stops.stop_lat.between(bbox.lat_min, bbox.lat_max) &
          feed.stops.stop_lon.between(bbox.lon_min, bbox.lon_max)
      ]
      ```
    - normalize() returns list[Observation] where:
      - domain="transit" for stops
      - values = {"stop_name": row.stop_name, "geometry_type": "Point", "feature_type": "stop", "lat": float(row.stop_lat), "lon": float(row.stop_lon)}
      - source_id = f"stop:{row.stop_id}"
      - feature_id is set to the string "PENDING" in normalize() — actual upsert happens in run()
    - normalize() ALSO processes shapes: for each shape_id that has at least one point within the Aalen bbox, build a WKT LineString from ordered shape points and return an Observation with domain="transit", values={"geometry_type": "LineString", "feature_type": "shape", "linestring_wkt": wkt}, source_id=f"shape:{shape_id}"
    - Override run() to: call fetch(), normalize() to get list of stop/shape observations with PENDING feature_ids, then bulk-upsert features via upsert_feature() (replace PENDING with real UUIDs), call _update_staleness() at end

        """Create a minimal valid GTFS zip in memory for unit tests.

        stops: list of dicts with stop_id, stop_name, stop_lat, stop_lon
        shapes: list of dicts with shape_id, shape_pt_lat, shape_pt_lon, shape_pt_sequence

        buf = io.BytesIO()
        with zipfile.ZipFile(buf, "w") as zf:
            # stops.txt
            stop_lines = ["stop_id,stop_name,stop_lat,stop_lon"]
            for s in stops:
                stop_lines.append(f"{s['stop_id']},{s['stop_name']},{s['stop_lat']},{s['stop_lon']}")

    - Override run() to handle async upsert_feature calls:
      ```python
      async def run(self) -> None:
          raw = await self.fetch()
          normalized = self.normalize(raw)  # returns list with PENDING feature_ids
          # upsert each feature and build real observations
          observations = []
          for item in normalized:
              geom_type = item.values["geometry_type"]
              if geom_type == "Point":
                  wkt = f"POINT({item.values['lon']} {item.values['lat']})"
              else:
                  wkt = item.values["linestring_wkt"]
              feature_id = await self.upsert_feature(
                  source_id=item.source_id,
                  domain="transit",
                  geometry_wkt=wkt,
                  properties={k: v for k, v in item.values.items() if k not in ("linestring_wkt",)},
              )
              observations.append(Observation(
                  feature_id=feature_id,
                  domain="transit",
                  values=item.values,
                  source_id=item.source_id,
              ))
          # GTFSConnector writes to features table only (static topology).
          # Do NOT call persist() — transit_positions is for GTFS-RT vehicle positions, not static stops.
          await self._update_staleness()
      ```
