  <behavior>
    GTFSConnector behavior:
    - fetch() downloads bwgesamt.zip from `self.config.config["gtfs_url"]` using httpx; returns raw bytes
    - fetch() uses `httpx.AsyncClient(timeout=120.0)` — GTFS zip is ~50MB
    - fetch() raises ValueError if response bytes are empty (len == 0)
    - normalize(raw: bytes) parses with `gtfs_kit.read_feed(io.BytesIO(raw), dist_units="km")`
    - normalize() IMMEDIATELY applies bbox filter after reading feed:
      ```python
      bbox = self.town.bbox
      stops = feed.stops[
          feed.stops.stop_lat.between(bbox.lat_min, bbox.lat_max) &
          feed.stops.stop_lon.between(bbox.lon_min, bbox.lon_max)
      ]
      ```
    - normalize() returns list[Observation] where:
      - domain="transit" for stops
      - values = {"stop_name": row.stop_name, "geometry_type": "Point", "feature_type": "stop"}
      - source_id = f"stop:{row.stop_id}"
      - feature_id is set to the string "PENDING" in normalize() — actual upsert happens in run()
    - normalize() ALSO processes shapes: for each shape_id that has at least one point within the Aalen bbox, build a WKT LineString from ordered shape points and return an Observation with domain="transit", values={"geometry_type": "LineString", "feature_type": "shape"}, source_id=f"shape:{shape_id}"
    - The run() method is overridden to: call fetch(), normalize() to get list of stop/shape dicts, then bulk-upsert features via upsert_feature(), then call persist() with the resulting Observations (with real feature UUIDs)

    Key implementation details:
    - Shape filtering: a shape is included if ANY of its points fall within the bbox (use: shape_pts grouped by shape_id, filter groups where any lat/lon is in bbox)
    - LineString WKT: `"LINESTRING({lon1} {lat1}, {lon2} {lat2}, ...)"` — note WKT uses (lon lat) order
    - Stops WKT: `"POINT({lon} {lat})"` — lon first (GeoJSON/WKT convention)
    - Do NOT hardcode stop count thresholds — the bbox filter from town.bbox drives what gets written
    - Attribution: store DL-DE-BY-2.0 attribution in feature properties: `{"attribution": "NVBW - Nahverkehrsgesellschaft Baden-Württemberg mbH, DL-DE-BY-2.0"}`

    Test behaviors:
    - test_gtfs_fetch_returns_bytes: GTFSConnector.fetch() returns bytes with len > 0
    - test_gtfs_normalize_filters_to_bbox: normalize(raw) returns only stops within Aalen bbox (no Stuttgart stops)
    - test_gtfs_stop_count_reasonable: number of stops is between 10 and 500 (sanity check for Aalen area)
    - test_gtfs_observations_have_point_geometry: stop Observations have values["geometry_type"] == "Point"
    - test_gtfs_shapes_are_linestrings: shape Observations have values["geometry_type"] == "LineString"

    Create `backend/app/connectors/gtfs.py`:
    - normalize() stores lat/lon in values dict for bbox verification in tests
    - Override run() to handle async upsert_feature calls:
      ```python
      async def run(self) -> None:
          raw = await self.fetch()
          normalized = self.normalize(raw)  # returns list with PENDING feature_ids
          # now upsert each feature and build real observations
          observations = []
          for item in normalized:
              geom_type = item.values["geometry_type"]
              if geom_type == "Point":
                  wkt = f"POINT({item.values['lon']} {item.values['lat']})"
              else:
                  wkt = item.values["linestring_wkt"]
              feature_id = await self.upsert_feature(
                  source_id=item.source_id,
                  domain="transit",
                  geometry_wkt=wkt,
                  properties={k: v for k, v in item.values.items() if k not in ("linestring_wkt",)},
              )
              observations.append(Observation(
                  feature_id=feature_id,
                  domain="transit",
                  values=item.values,
                  source_id=item.source_id,
              ))
          # transit stops/shapes don't write time-series rows; persist() will no-op for transit domain
          # (transit_positions is for RT vehicle positions, not static features)
          # So no await self.persist(observations) needed here — upsert_feature handles the DB write
      ```

    Note: GTFSConnector writes to the `features` table (spatial features), NOT to the `transit_positions` hypertable (which is for GTFS-RT vehicle positions/delays). The persist() method for "transit" domain in BaseConnector writes to transit_positions. GTFSConnector should NOT call persist() — it only calls upsert_feature() to write static spatial data.

    Shape WKT construction:
    ```python
    # Group shapes by shape_id, sort by shape_pt_sequence
    for shape_id, group in feed.shapes.groupby("shape_id"):
        pts = group.sort_values("shape_pt_sequence")
        # Check if any point is in bbox
        in_bbox = (
            pts.shape_pt_lat.between(bbox.lat_min, bbox.lat_max) &
            pts.shape_pt_lon.between(bbox.lon_min, bbox.lon_max)
        ).any()
        if not in_bbox:
            continue
        coords = " , ".join(f"{row.shape_pt_lon} {row.shape_pt_lat}" for _, row in pts.iterrows())
        wkt = f"LINESTRING({coords})"
        # store as Observation with values={"geometry_type": "LineString", "linestring_wkt": wkt, "feature_type": "shape"}
    ```
