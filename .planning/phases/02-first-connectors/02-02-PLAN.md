```python
@dataclass
class Observation:
    feature_id: str   # UUID string from features table
    domain: str       # "air_quality"
    values: dict      # {"pm10": float|None, "pm25": float|None, "no2": float|None, "o3": float|None, "aqi": float|None}
    timestamp: datetime | None = None
    source_id: str | None = None

class BaseConnector(ABC):
    def __init__(self, config: ConnectorConfig, town: Town) -> None: ...
    async def fetch(self) -> Any: ...
    def normalize(self, raw: Any) -> list[Observation]: ...
    async def run(self) -> None: ...  # fetch -> normalize -> persist
    async def persist(self, observations: list[Observation]) -> None: ...
    async def upsert_feature(self, source_id, domain, geometry_wkt, properties) -> str: ...
```

  <behavior>
    UBAConnector behavior:
    - fetch() calls `https://luftdaten.umweltbundesamt.de/api-proxy/airquality/json` with station param from `self.config.config["station_id"]` (integer, e.g. 238)
    - fetch() raises ValueError if response["data"] is empty or missing
    - normalize() creates UBAMeasurement Pydantic models for each time entry; handles missing component IDs gracefully (not every station measures all pollutants)
    - normalize() maps component_id to pollutant name (PM10=1, NO2=5, O3=3, SO2=2, CO=4, PM2.5=9)
    - normalize() calls upsert_feature() with "POINT(10.09 48.84)" (from config.config["lon"]/config.config["lat"]) to get feature_id UUID
    - normalize() returns list[Observation] with domain="air_quality"
    - All pollutant values in Observation.values are float | None — never raise KeyError

    SensorCommunityConnector behavior:
    - fetch() calls `https://data.sensor.community/airrohr/v1/filter/` with area param computed from self.town.bbox center + radius 25km
    - fetch() MUST set User-Agent header: "city-data-platform/0.1 (open-source city dashboard)"
    - fetch() raises ValueError if response list is empty
    - normalize() filters for sensors with sensor_type.name in ("SDS011", "SPS30") — skip other types
    - normalize() extracts P1 (PM10) and P2 (PM2.5) from sensordatavalues; missing values become None
    - normalize() calls upsert_feature() with "POINT({lon} {lat})" from sensor location for each sensor
    - Each sensor.id is used as source_id in features upsert
    - Returns list[Observation] with domain="air_quality"

    UBAMeasurement Pydantic model:
    - Fields: station_id: int, component_id: int, date_end: datetime, value: float | None, index: int | None
    - field_validator("value"): if v is not None and v < 0: return None (reject negative readings)

    SensorReading Pydantic model:
    - Fields: sensor_id: int, sensor_type: str, lat: float, lon: float, pm10: float | None, pm25: float | None
    - Created by parsing the sensordatavalues array: P1 -> pm10, P2 -> pm25

    - test_uba_normalize_returns_observations: normalize() returns list with at least one Observation where domain="air_quality"
    - test_uba_no_negative_values: all float values in Observation.values are >= 0 or None
    - test_sensor_community_fetch_returns_sensors: SensorCommunityConnector.fetch() returns list with len > 0
