    1. Add GET /features/search endpoint to backend/app/routers/features.py:
       - Query param: q (string, min 2 chars)
       - Query param: town (string, required)
       - Query param: limit (int, default 10, max 50)
       - SQL: Search features table using ILIKE on properties->>'name', properties->>'address', semantic_id, source_id
         ```sql
         SELECT id::text, semantic_id, domain, source_id,
                ST_AsGeoJSON(geometry)::text AS geometry,
                properties
         FROM features
         WHERE town_id = :town_id
           AND (
             properties->>'name' ILIKE :q
             OR properties->>'address' ILIKE :q
             OR semantic_id ILIKE :q
             OR source_id ILIKE :q
           )
         LIMIT :limit
         ```
       - Return array of { id, semantic_id, domain, name, geometry: {lat, lon} }
       - Name extracted from properties->>'name' or properties->>'stop_name' or source_id as fallback
       - IMPORTANT: Route path must be /features/search (not /features/{feature_id}/search) to avoid path conflict with the {feature_id} route. Place search route BEFORE the {feature_id} route in the router.

       - FeaturePopup (air quality / transit): Same pattern
       - Pattern for all: Only show the cross-domain section if useFeatureData returns observations in domains OTHER than the popup's own domain. If no cross-domain data, show nothing extra.
