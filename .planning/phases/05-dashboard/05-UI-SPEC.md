## Typography

Identical scale to Phase 4 — dashboard adds one display size for KPI values.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 16px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.1 |

Notes:
- Display (28px/600): KPI tile primary metric value only (e.g., "42 µg/m³", "94%")
- Heading (16px/600): KPI tile domain label, detail panel section headers, chart titles
- Body (14px/400): chart axis labels, date range picker text, detail panel body text, time slider timestamp
- Label (12px/400): KPI tile unit suffix (µg/m³, km/h, %) shown below value, trend arrow percentage, attribution text
- All text renders in German. No i18n layer in Phase 5.

## Layout

Source: CONTEXT.md ("Right panel alongside map — split view (left=map, right=dashboard)", "Time slider below the map — horizontal scrubber, full width").

### Desktop (≥ 1024px)

```
┌────────────────────────────────────────────────────────────────┐
│ Sidebar (280px) │ Map (flex-grow)              │ Dashboard (320px) │
│ ─────────────── │                              │ ──────────────── │
│ [Phase 4 as-is] │  MapLibre GL map             │ KPI TILES        │
│                 │                              │ [AQI tile]       │
│                 │                              │ [Weather tile]   │
│                 │                              │ [Transit tile]   │
│                 │ ─────────────────────────── │ ──────────────── │
│                 │ [Time Slider — full width]   │ DATE RANGE       │
│                 │                              │ [24h][7d][30d][custom] │
│                 │                              │ ──────────────── │
│                 │                              │ CHART AREA       │
│                 │                              │ [Line/area chart]│
└────────────────────────────────────────────────────────────────┘
```

- Left sidebar: 280px fixed (Phase 4, unchanged)
- Map: flex-grow, fills remaining width between sidebar and dashboard panel
- Dashboard panel: 320px fixed, right side, full viewport height, scrollable internally
- Time slider: rendered BELOW the map canvas, full width of the map column (not spanning dashboard panel), height 48px

### KPI Tile → Detail Panel Expand

When a KPI tile is clicked:
- The chart area below the KPI tiles is replaced by a domain-specific detail panel
- Detail panel fills the same 320px column below the KPI tiles
- A back chevron ("← Zurück") returns to the default chart view
- No modal or overlay — in-place replacement only

### Tablet (768px – 1023px)

- Sidebar collapses to drawer (Phase 4 pattern unchanged)
- Dashboard panel collapses to a drawer triggered by a floating button (bottom-right, above time slider)
- Map occupies full viewport
- Time slider remains pinned at bottom, full viewport width
- No horizontal scroll

### Mobile (< 768px)

Not the primary target for Phase 5. Layout must not break — drawers stack. No Phase 5-specific mobile optimizations.

## Interaction Contracts

### KPI Tiles

Default state: 3 tiles visible in dashboard panel — AQI, Weather, Transit.

| Domain | Icon | Primary Metric | Unit | Trend |
|--------|------|----------------|------|-------|
| Air Quality | `wind` (lucide) | Current AQI index value | "AQI" | vs. 24h ago |
| Weather | `thermometer` (lucide) | Current temperature | "°C" | vs. yesterday |
| Transit | `bus` (lucide) | GTFS routes loaded | "Linien" | static (no trend arrow) |

Click behavior:
- Clicking a KPI tile sets `activeDomain` state to that domain
- Chart area below KPI tiles renders `DomainDetailPanel` for the active domain
- Tile border ring activates (accent color) to show selected state
- Clicking the same tile again collapses detail panel back to default chart view

Tile hover: secondary background (#f7f7f7) — CSS `hover:bg-secondary` from shadcn

### Date Range Picker

Preset buttons: "24h", "7d", "30d" as shadcn Button (variant="outline", active state: variant="default")

Custom range:
- Clicking "Zeitraum" opens a shadcn Popover containing a shadcn Calendar in range-select mode
- Selected range displayed as "DD.MM.YYYY – DD.MM.YYYY" in the button label
- Applying range closes the popover; charts re-fetch with new `?start=&end=` params
- Max selectable range: 90 days (beyond that, continuous aggregates are insufficient)

Date range state is encoded in URL search params: `?from=YYYY-MM-DD&to=YYYY-MM-DD`

### Time Series Charts

Default chart per domain:
- Air Quality: AreaChart — PM2.5 and PM10 as two overlaid areas (chart-2 and chart-4 tokens)
- Weather: LineChart — temperature over time (chart-2)
- Transit: Not charted (static GTFS data, no time-series)

Chart behavior:
- X axis: time labels at appropriate interval for selected range (hourly for 24h, daily for 7d/30d)
- Y axis: left-side, no units in tick labels (unit shown in chart title)
- Tooltip on hover: shows exact timestamp (DD.MM.YYYY HH:mm) + value + unit
- No animation on initial mount (reduces perceived latency)
- Loading state: skeleton shimmer replacing chart area (Tailwind `animate-pulse` on placeholder div)
- Empty data: empty state text (see Copywriting)
- Error state: error text with retry instruction (see Copywriting)

### Time Slider

Position: pinned below map canvas, above viewport bottom, full map-column width (not extending into dashboard panel).

Behavior:
- Range: from 24 hours ago to "now"
- Default position: rightmost (current time / live data)
- Scrubbing: dragging the handle sends `historicalTimestamp` to MapView — map layers query the API at that timestamp
- Charts do NOT respond to the time slider — the date range picker controls chart window
- At rightmost position (live): map layers use real-time data (no timestamp param)
- Timestamp label above the slider handle shows selected time as "HH:mm" when scrubbing, hidden at rest
- Step resolution: 15 minutes (matches APScheduler polling intervals from CLAUDE.md)

Slider state is encoded in URL: `?ts=ISO8601` when not at live position, omitted when live.

### Permalink / URL State

Full URL state encoding (extends Phase 4 where layer state was not encoded):

| Param | Example | Source |
|-------|---------|--------|
| `town` | `aalen` | town config |
| `layers` | `transit,aqi` | active layer toggles |
| `zoom` | `13` | map viewport |
| `lat` | `48.8374` | map viewport |
| `lng` | `10.0918` | map viewport |
| `from` | `2026-04-04` | date range picker |
| `to` | `2026-04-05` | date range picker |
| `ts` | `2026-04-05T14:00:00Z` | time slider (omitted when live) |
| `domain` | `aqi` | active detail panel (omitted when none) |

URL is updated via `window.history.pushState` (or Next.js `useSearchParams` + `router.replace`) on every state change. Pasting URL restores exact view on mount.

Source: CONTEXT.md ("URL state via search params: ?town=aalen&layers=transit,aqi&zoom=13&lat=48.84&lng=10.09&from=...&to=...")

### Data Refresh

- KPI tiles: poll `/api/kpi?town=aalen` every 60 seconds
- Chart data: fetched once on mount + on date range change; no auto-refresh (historical data is stable)
- Time slider historical queries: fetched on scrub-end (not on every drag tick — debounced 300ms)
- During refetch: existing chart data remains visible with a subtle loading spinner in chart title area
