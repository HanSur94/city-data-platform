       ```python
           "air_quality": {"yellow": 3600, "red": 7200},      # 1h/2h — sensors report frequently

           "demographics": {"yellow": 604800, "red": 2592000}, # 7d/30d — slow-moving data
       }
       ```

    2. AdminHealthItem response model (extends ConnectorHealthItem):
       ```python
       class AdminHealthItem(BaseModel):
           id: str
           domain: str
           connector_class: str
           last_successful_fetch: datetime | None
           validation_error_count: int
           status: str        # "green", "yellow", "red", "never_fetched"
           staleness_seconds: float | None  # seconds since last fetch
           poll_interval: int | None        # configured poll interval
           threshold_yellow: int
           threshold_red: int

       class AdminHealthResponse(BaseModel):
           town: str
           town_display_name: str
           checked_at: datetime
           summary: dict[str, int]  # {"green": N, "yellow": N, "red": N, "never_fetched": N}
           connectors: list[AdminHealthItem]
       ```

    3. The endpoint logic:
       - Query sources table for all connectors (same as connectors.py)
       - For each connector, compute staleness_seconds = (now - last_successful_fetch).total_seconds()
       - Apply domain-specific thresholds: green if < yellow threshold, yellow if < red threshold, red if >= red threshold
       - Include poll_interval from town config by matching connector_class
       - Build summary counts
       - Return AdminHealthResponse

    4. Register router in backend/app/main.py:
       ```python
       from app.routers import admin
       app.include_router(admin.router, prefix="/api")
       ```
