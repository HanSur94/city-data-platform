    Create backend/app/models/bus_interpolation.py with Pydantic models:
    - BusPosition: trip_id, route_id, line_name, destination, next_stop, lat, lon, bearing, delay_seconds, progress (0.0-1.0), departed (bool)
    - ActiveTrip: trip_id, route_id, line_name (route_short_name), destination (trip_headsign or last stop name), stop_times list, shape_coords list, delay_seconds

    Create backend/app/connectors/bus_interpolation.py:
    - BusInterpolationConnector inherits BaseConnector
    - Override run() (does NOT use standard fetch/normalize/persist pipeline since this is a computation connector)
    - run() flow:
      1. Download/cache GTFS zip (reuse gtfs_url from aalen.yaml GTFSConnector config, cache in /tmp for 24h)
      2. Parse with gtfs_kit: extract trips, stop_times, routes, shapes, calendar/calendar_dates
      3. Determine active service_ids for today using calendar + calendar_dates
      4. Find active trips: trips whose service_id is active AND whose first departure <= now <= last arrival (adjusted by delay)
      5. For each active trip, fetch GTFS-RT delay (query transit_positions table for latest delay_seconds by trip_id from GTFSRealtimeConnector data)
      6. Calculate position: find current segment between stops, compute progress fraction, walk along shape geometry
      7. Upsert feature with source_id="bus-pos:{trip_id}", domain="transit", geometry=interpolated POINT, properties={feature_type: "bus_position", trip_id, route_id, line_name, destination, next_stop, delay_seconds, bearing, progress}
      8. Write to transit_positions hypertable via persist() for trend tracking
      9. Clean up stale bus-pos features (trips that are no longer active)

    Shape-walking algorithm (pure function, easily testable):
    - Input: shape_coords (list of (lon, lat) tuples), progress (0.0-1.0)
    - Compute cumulative distances along shape using haversine
    - Find target distance = progress * total_distance
    - Walk segments until target distance is reached
    - Linear interpolation within the target segment
    - Also compute bearing from interpolated point to next point

    Edge cases per REQ-BUS-05:
    - Dwelling at stop: if current time is between arrival and departure at same stop, position = stop coords
    - No delay data: assume delay_seconds=0 (pure schedule)
    - Trip not departed: current time < first_departure + delay -> show at first stop, set departed=false
    - Trip completed: current time > last_arrival + delay -> skip (do not upsert feature)
    - Multiple buses on same route: each tracked by unique trip_id

    IMPORTANT: The connector reads the GTFS-RT delay data from the database (transit_positions written by GTFSRealtimeConnector), NOT by fetching GTFS-RT protobuf directly. This avoids duplicate fetching.

    Use asyncio.to_thread for gtfs_kit parsing (it is synchronous/blocking), same pattern as Phase 12 LhpConnector.
