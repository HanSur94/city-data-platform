  <action>
    Create `backend/scripts/backfill_aqi.py` as a standalone async script:

    ```python
    #!/usr/bin/env python3
    """Backfill 90 days of UBA air quality data for station 238 (Aalen DEBW029).

    Usage (from backend/ directory):
        uv run python scripts/backfill_aqi.py [--days 90] [--station 238]

    Fetches UBA historical data in 7-day chunks, normalizes with UBAConnector.normalize(),
    and inserts directly into air_quality_readings. Skips duplicates via ON CONFLICT DO NOTHING.
    """
    ```

    Script structure:
    1. Parse CLI args: `--days` (default 90), `--station` (default 238)
    2. Compute date range: `end = date.today()`, `start = end - timedelta(days=days)`
    3. Chunk loop: iterate in 7-day windows from start to end
    4. For each chunk: fetch UBA API with `date_from`/`date_to`, normalize with UBAConnector.normalize()-equivalent logic (can inline the parsing since it's a script)
    5. For each Observation, compute EAQI:
       ```python
       from app.schemas.geojson import eaqi_from_readings
       tier_idx, _, _ = eaqi_from_readings(pm25=obs.values.get("pm25"), ...)
       ```
    6. INSERT with ON CONFLICT DO NOTHING (safe to re-run):
       ```sql
       INSERT INTO air_quality_readings (time, feature_id, pm25, pm10, no2, o3, aqi)
       VALUES (:time, :feature_id, :pm25, :pm10, :no2, :o3, :aqi)
       ON CONFLICT DO NOTHING
       ```
    7. Feature upsert: look up or create the feature for station 238 using the same
       `upsert_feature` SQL pattern (inline, not via connector) with:
       - source_id = "uba:238", domain = "air_quality", geometry = "POINT(10.093 48.836)"
       - properties = {"station_id": 238, "attribution": "Umweltbundesamt (UBA)"}
    8. Print progress: `Chunk YYYY-MM-DD → YYYY-MM-DD: N observations inserted`
    9. Print summary: `Total: N observations inserted across 90 days`

    Use `asyncio.run(main())` entry point. Use `httpx.AsyncClient` for fetching.
    Use `sqlalchemy.ext.asyncio.create_async_engine` with `DATABASE_URL` env var.
    Add 0.5s sleep between chunks to be polite to UBA API.

    NOTE: This script is run once manually after deploying Phase 6. It is not a
    scheduled connector. Document the run command in a docstring.
