```
# Naturschutzgebiet (nature protection zones):
GET https://rips-gdi.lubw.baden-wuerttemberg.de/arcgis/services/wfs/Naturschutzgebiet/MapServer/WFSServer
    ?service=WFS&version=2.0.0&request=GetFeature
    &typeNames=Naturschutzgebiet
    &outputFormat=application/json
    &srsName=urn:ogc:def:crs:EPSG::4326
    &BBOX=48.76,9.97,48.90,10.22,urn:ogc:def:crs:EPSG::4326

# Wasserschutzgebiet (water protection zones):
GET https://rips-gdi.lubw.baden-wuerttemberg.de/arcgis/services/wfs/Wasserschutzgebiet/MapServer/WFSServer
    ?service=WFS&version=2.0.0&request=GetFeature
    &typeNames=Wasserschutzgebiet
    &outputFormat=application/json
    &srsName=urn:ogc:def:crs:EPSG::4326
    &BBOX=48.76,9.97,48.90,10.22,urn:ogc:def:crs:EPSG::4326

# CRITICAL: Default CRS is EPSG:25832 (UTM). Request 4326 via srsName.
# BBOX for aalen: lon_min=9.97, lat_min=48.76, lon_max=10.22, lat_max=48.90
# WFS may return 0 features if no zones overlap bbox — that is valid (not an error).
```

WFS GeoJSON response shape:
```json
{
  "type": "FeatureCollection",
  "features": [{
    "type": "Feature",
    "id": "Naturschutzgebiet.123",
    "geometry": {"type": "Polygon", "coordinates": [...]},
    "properties": {
      "NSG_NR": "1.131",
      "NAME": "Welland",
      "FLAECHE_HA": 42.5
    }
  }]
}
```

BaseConnector.upsert_feature() signature:
```python
async def upsert_feature(
    self,
    source_id: str,       # e.g. "naturschutz:1.131" or "wasserschutz:WSG-123"
    domain: str,          # "water" (VALID_DOMAINS includes "water")
    geometry_wkt: str,    # WKT representation; for polygons use ST_GeomFromGeoJSON instead
    properties: dict,
) -> str                  # returns UUID
```

NOTE: upsert_feature() uses ST_GeomFromText(wkt, 4326). For polygon features, the geometry
must be converted to WKT. Use shapely (already available via geopandas dependency) or
construct a GeoJSON-aware INSERT directly.

Alternative approach — override the geometry insert to use ST_GeomFromGeoJSON:
Since upsert_feature() only supports WKT, for polygon geometries use the
centroid POINT as geometry (adequate for map display of zone boundaries).
The full polygon shape can be a WMS overlay (Plan 04). For WATR-05, storing
the centroid feature enables the layer endpoint to return the feature with
name/type properties even if the polygon shape itself comes from WMS.

If shapely IS available (check pyproject.toml), use:
```python
from shapely.geometry import shape
from shapely.wkt import dumps as to_wkt
geom = shape(feature["geometry"])
wkt = to_wkt(geom)  # proper WKT for ST_GeomFromText
```

If shapely is NOT available, use centroid approach:
```python
# For polygons: compute centroid manually or use GeoJSON ring average
coords = feature["geometry"]["coordinates"][0]
lon = sum(c[0] for c in coords) / len(coords)
lat = sum(c[1] for c in coords) / len(coords)
wkt = f"POINT({lon} {lat})"
```
