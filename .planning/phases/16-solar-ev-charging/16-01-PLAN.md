Purpose: Backend data engine for REQ-SOLAR-01 (solar production computation) and REQ-EV-01/02/03 (live EV charger status)

From backend/app/connectors/base.py:
```python
class Observation:
    feature_id: str
    domain: str       # "energy" for solar, "infrastructure" for EV
    values: dict      # {"value_kw": ..., "source_type": "solar_production"} or EV status props
    timestamp: datetime | None
    source_id: str | None

class BaseConnector(ABC):
    def __init__(self, config: ConnectorConfig, town: Town) -> None
    async def fetch(self) -> Any
    def normalize(self, raw: Any) -> list[Observation]
    async def run(self) -> None  # fetch -> normalize -> persist -> _update_staleness
    async def upsert_feature(self, source_id, domain, geometry_wkt, properties) -> str
    async def _update_staleness(self) -> None
```

Energy readings INSERT schema (from base.py persist):
```python
"INSERT INTO energy_readings (time, feature_id, value_kw, source_type) VALUES (:time, :feature_id, :value_kw, :source_type) ON CONFLICT DO NOTHING"
```

MaStR features are stored with domain="energy", properties include:
- installation_type: "solar_rooftop" | "solar_ground" | "wind" | "battery"
- capacity_kw: float (net capacity in kW)

Infrastructure features are stored with domain="infrastructure", category="ev_charging"

Bright Sky current_weather API returns (among other fields):
- cloud_cover: float (0-100, percent)
- sunshine_duration: float (minutes in last period)
- solar: float (J/cm2 in last 10 minutes — solar irradiance)
Note: The Bright Sky API field "solar" provides global irradiance. Use this if available, else estimate from cloud_cover.

MobiData BW OCPDB API:
- Endpoint: https://api.ocpdb.2hat.de/api/ocpi/2.2/location
- No auth required (public API)
- Returns OCPI 2.2 location objects with EVSEs and connectors
- Each EVSE has status: AVAILABLE | CHARGING | INOPERATIVE | OUT_OF_ORDER | UNKNOWN
- Each connector has power_type, max_electric_power (W)
- Filter by coordinates/bounding box

       - SolarInstallation(BaseModel): feature_id (str), capacity_kw (float), lon (float), lat (float)

    2. Create backend/app/connectors/solar_production.py:
       - class SolarProductionConnector(BaseConnector) -- computation connector pattern (like AirQualityGridConnector)
       - Pure function compute_irradiance_factor(cloud_cover: float | None, solar_j_cm2: float | None) -> float:
         * If solar_j_cm2 is available and > 0: factor = min(solar_j_cm2 / 30.0, 1.0)
           (30 J/cm2 per 10 min ~ 500 W/m2 clear-sky summer reference for central Europe)
         * Else if cloud_cover is available: factor = max(1.0 - cloud_cover / 100.0 * 0.9, 0.1)
           (0% cloud -> 1.0, 100% cloud -> 0.1 floor for diffuse radiation)
         * Else: return 0.5 (fallback)
       - Pure function compute_solar_output(capacity_kw: float, irradiance_factor: float) -> float:
         * return capacity_kw * irradiance_factor
       - Override run() to:
         a. Fetch current weather from Bright Sky API (https://api.brightsky.dev/current_weather?lat={lat}&lon={lon}) to get cloud_cover and solar fields
         b. Compute irradiance_factor from weather data
         c. Query all solar installations from features table: SELECT id, properties->>'capacity_kw', ST_X(geometry), ST_Y(geometry) FROM features WHERE town_id=self.town.id AND domain='energy' AND (properties->>'installation_type' LIKE 'solar_%')
         d. For each installation, compute current_output_kw = compute_solar_output(capacity_kw, factor)
         e. Create Observation(feature_id=installation.id, domain="energy", values={"value_kw": current_output_kw, "source_type": "solar_production"})
         f. Call self.persist(observations)
         g. Update feature properties with current_output_kw for frontend display: UPDATE features SET properties = properties || '{"current_output_kw": X, "irradiance_factor": Y}' WHERE id = installation.id
         h. Call self._update_staleness()
       - Do NOT use fetch/normalize pattern -- override run() directly (computation connector)
       - Use lat/lon from self.config.config for weather query (same as WeatherConnector)

    3. Create backend/tests/connectors/test_solar_production.py:
       - Test the pure functions (compute_irradiance_factor, compute_solar_output) directly
       - No database needed for pure function tests
