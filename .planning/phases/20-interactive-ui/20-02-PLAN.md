  <action>
    Create `DataExplorerModal.tsx` — a floating, draggable, resizable modal for data exploration.

    **Do NOT use react-rnd** — implement with native CSS resize and pointer events for drag. This avoids adding a dependency.

    **Component structure:**

    ```
    interface DataExplorerModalProps {
      domain: string            // 'aqi' | 'weather' | 'transit' | 'traffic' | 'energy' | 'demographics' | 'parking'
      town: string
      onClose: () => void
      initialPosition: { x: number; y: number }  // viewport coordinates
    }
    ```

    **Drag implementation:**
    - Track `position` state `{ x: number, y: number }` initialized from `initialPosition`.
    - On the title bar div: `onPointerDown` sets a `dragging` ref to true, stores the offset between pointer and position.
    - Attach `pointermove` and `pointerup` listeners to `window` during drag.
    - On `pointermove`: update position = pointer - offset.
    - On `pointerup`: set dragging to false, remove listeners.
    - Use `style={{ transform: \`translate(\${x}px, \${y}px)\` }}` on the outer container (position: fixed, top: 0, left: 0).

    **Resize implementation:**
    - Use CSS `resize: both; overflow: auto` on the content container.
    - Set `min-width: 360px; min-height: 300px; max-width: 90vw; max-height: 80vh`.

    **Visual design:**
    - Outer container: `fixed z-50 bg-white rounded-lg shadow-2xl border border-slate-200`.
    - Default size: `w-[480px] h-[400px]`.
    - Title bar: `flex items-center justify-between px-4 py-2 bg-slate-50 rounded-t-lg border-b cursor-grab active:cursor-grabbing select-none`.
    - Title: domain label (use DOMAIN_LABELS map: aqi -> "Luftqualitaet", weather -> "Wetter", transit -> "OEPNV", traffic -> "Verkehr", energy -> "Energie", demographics -> "Demografie", parking -> "Parken").
    - Close button: `X` icon from lucide-react, 20px, text-slate-400 hover:text-slate-600.
    - Content area: `flex-1 overflow-auto p-4`.

    **Content per domain:**
    - Reuse the same rendering logic as DomainDetailPanel but without the back button.
    - Use `useKpi(town)` for latest values and `useTimeseries(domain, from, to, town)` for chart data.
    - Default date range: last 24 hours (from = new Date(Date.now() - 86400000), to = new Date()).
    - For `transit`: show static info ("OEPNV-Daten sind statisch...").
    - For `demographics`: show the StatCard grid from DomainDetailPanel.
    - For `aqi`, `weather`: show TimeSeriesChart.
    - For `traffic`, `energy`: show the respective chart from DomainDetailPanel (line chart for traffic, stacked area for energy).
    - For `parking`: show latest values from KPI (total_free, total_capacity, availability_pct) as StatCard-style items.

    **Key implementation details:**
    - Use `'use client'` directive.
    - Use `useEffect` cleanup to remove window event listeners on unmount.
    - Use `setPointerCapture` on the title bar during drag for reliable tracking.
  </action>
